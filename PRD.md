# 夢守り（ゆめまもり）- 製品要件定義書 (PRD)

## 1. 概要 (Overview)
「夢守り（ゆめまもり）」は、ユーザーが朝目覚めたときに覚えている夢の断片を記録し、AIパートナーのサポートを得て自己理解を深めるための**夢セルフケア・バランシングアプリケーション**です。
本アプリは、夢の吉凶を占ったり、精神分析的なラベルを貼って断定・診断したりするものではありません。夢を客観的かつ温かく整理し、心身の余白を保つための具体的なセルフケアの提案、そして自らの深層意識を美しいビジュアルとして愛でる「夢の庭」を提供することで、朝の静かなセルフコンディショニングを促します。

---

## 2. コアコンセプト & 思想 (Core Philosophies)
* **占いや診断ではなく、内省と受容**: 「〇〇の夢はストレスのサインです」「トラブルの予兆です」といった不安を煽る断定は行いません。ありのままの気配を構造化し、ユーザー自身が「そうかもしれない」と主体的に気づくための余白を提供します。
* **安全を最優先にする（グラウンディング設計）**: 悪夢やパニック状態、精神的な危機（自傷・他害・希死念慮など）を検知した場合は即座に分析を安全モードに切り替え、深い探索を中止して深呼吸や体を温めるセルフケアへ誘導します。
* **無意識の可視化としての「夢の庭」**: 言語化されたデータだけでなく、夢から生まれた象徴や場所が自分だけの「庭（インナーガーデン）」として少しずつ成長・蓄積されていくことで、自らの内面世界を優しく客観視することができます。

---

## 3. 主要機能一覧 (Core Features)

### ① 夢の記録と入力 (Dream Capture)
* **テキスト入力 & 音声入力シミュレーション**: 朝の微睡みの中でも簡単に入力できるよう、キーボード入力に加え、マイクを用いた音声録音（文字起こしシミュレーション）に対応。
* **ヒントチップ（誘い水）**: 「何から逃げていた？」「何色だった？」「どんな感覚が残っている？」など、あやふやな夢の記憶を呼び起こすトリガーワードを表示。
* **3つのAI解析アプローチ選択（UI連携）**:
  * **標準やさしい整理モード (Default)**: 感情やモチーフを客観的に優しく分解して整理し、今の心の状態をありのままに受け止めます。
  * **本音をひもとくモード (フロイト派/Freud)**: 夢を「無意識の願望や抑圧された葛藤の表現」と捉え、現実生活で我慢していることや隠れた本当の欲求にアプローチします。
  * **バランスをみつめるモード (ユング派/Jung)**: 夢を「心のバランスを修復するためのアドバイス（自己補償作用）」と捉え、普段見落としがちな内なるもう一人の自分の声に焦点を当てます。

### ② AI夢守り診断・分析 (Gemini AI Processing Engine)
* **モデル**: `gemini-3.5-flash` を利用。
* **レスポンス仕様**: 構造化されたJSONデータを安全にパース。
  * `dream_title`: 夢の内容を象徴する美しい15文字以内のタイトル。
  * `summary`: 夢のやさしい要約（150文字程度）。
  * `places / characters / emotions / motifs / colors`: 夢に現れたシンボル・気配の分類。
  * `metaphor`: 夢から感じ取れる、心のありようを表す詩的な解釈。
  * `reflection_question`: ユーザーへ問いかける、やさしい内省の質問。
  * `inner_weather`: 今日の心の天気予報（晴れ、霧、嵐、夜明け前など）と過ごし方のアドバイス。
  * `self_care`: 具体的な「今日できる小さなケア」（例：ハーブティーを飲む、スマホを1時間切るなど）。
  * `garden_update`: 夢から生まれる庭の場所・植物とその説明。

### ③ 夢の庭 (The Dream Garden - GardenCanvas)
* **インタラクティブな深層庭園**: 夢を記録するたびに、夢の要素（古い温室、静かな入り江、記憶の部屋など）が庭に追加されます。
* **成長システム**: 同じ場所やモチーフが何度も夢に出現すると、`appearance_count`（出現回数）が増え、庭の中の該当アイテムが大きく、あるいは光を増して成長します。
* **ビジュアル表現**: Tailwindとモーション、およびSVGフィルターを用いて、深い眠りと幻想的な内面世界を表現する有機的なアニメーション（ホバーエフェクト、微細な波打ち、漂う光の粒子など）を実装。

### ④ 今週のふりかえり (Weekly Review)
* **過去7日間の自動集計**:
  * 夢の回数、優勢だった感情・モチーフのトップランキング。
  * 心の天気の歴史（移り変わり）をタイムライン表示。
* **Geminiによる複合的な今週の振り返り**:
  * 蓄積された複数の夢ログから、1週間全体の共通メタファー（例：「立ち止まり、ゆっくりと次の季節を待つ森」）を創出。
  * 来週に向けた、中長期的なやさしいバランシングセルフケアを提案。

### ⑤ 記録一覧・振り返り (History & Archive)
* これまでに記録した全ての夢を、時系列カード形式で一覧表示。
* 各カードには分析時に選択した「標準」「フロイト派」「ユング派」などのアプローチバッジを表示。
* 不要になった、または手放したい夢を選択して削除可能（削除された夢のデータは庭の育成状態にも自動で再計算・反映されます）。

## 4. プロンプト設計 (System Instructions & Prompt Patterns)

### 夢解析プロンプト (`/api/dreams` 用)
本アプリでは、選択された分析モードによってシステム指示（`systemInstruction`）に動的なプロンプトが差し込まれ、精度の高いモード別のアプローチを実現しています。

#### ① モード選択に応じた動的インジェクション (`modeSpecificInstruction`)
```ts
// 本音をひもとくモード（フロイト派アプローチ）
【分析モード：本音をひもとくモード（フロイト派アプローチ）】
思想：夢は「抑圧された願望・本音の変装」である（過去・原因論）。
アプローチ：夢のシンボルやできごとを「本当は現実で何を我慢しているのか？」「何から目を背けたいのか？」という、ユーザーの無意識にある隠れた本音・葛藤や欲求の解放として読み解いてください。
- metaphor: 夢のシンボルを「抑圧された欲求や本音のシンボル」として読み解き、「あなたの心は今、本当は〜を求めているのかもしれません」という無意識の願いを優しく提示します。
- reflection_question: 「あなたがいま、現実で少し我慢していることや、本当は手に入れたいと感じているものは何でしょうか？」など、自分の隠れた本音・欲求に優しく目を向ける内省の質問にしてください。

// バランスをみつめるモード（ユング派アプローチ）
【分析モード：バランスをみつめるモード（ユング派アプローチ）】
思想：夢は「偏った心のバランスを直すためのアドバイス」である（未来・目的論）。
アプローチ：夢のシンボルや登場人物を「人類共通のキャラクター（原型）」や「今の自分の意識に足りない要素（影、アニマ／アニムスなど）」として捉え、これからの心の調和、自己実現へのステップとして読み解いてください。
- metaphor: 夢を「偏った心のバランスを補償し、調和を取り戻すための無意識からのアドバイス」として読み解き、「この夢は、あなたの中にあるもう一つの側面（〜な性質など）に気づき、統合していくためのメッセージかもしれません」と提示します。
- reflection_question: 「あなたが普段の生活で少し遠ざけている、自分の中のもう一人の自分は、どんな声をかけてくれているでしょうか？」など、心全体の調和を促す前向きな問いかけにしてください。

// 標準やさしい整理モード
【分析モード：標準やさしい整理モード】
アプローチ：夢の感情やモチーフを客観的に優しく分解して整理し、今の心の状態をありのままに受け止めます。
```

#### ② ベースのシステムプロンプト & 安全設計
```ts
あなたは「夢守り（ゆめまもり）」という夢 of セルフケアアプリのAIパートナーです。
ユーザーが朝、覚えている夢の断片（テキストや音声文字起こし）を入力します。それをやさしく受け止め、占いや断定的な診断ではなく、内省性と今日の小さなセルフケアのために構造化・言語化してください。

【振る舞い原則】
1. 断定しない: 「〜を意味します」ではなく、「〜に近い感覚が含まれているかもしれません」「〜という心の声が隠れている可能性があります」と言いましょう。
2. 診断しない: 「ストレスが溜まっています」などと断定せず、「夢の中に”急ぐ”や”逃げる”といったモチーフが含まれており、少しお疲れのサインかもしれません」とやさしく添えます。
3. 不安を煽らない: 怖い夢 or 悪夢に対しても、それが危険な予兆だと言ったりせず、心が記憶を整理しようとしているやさしい反応であると伝えてください。
4. 詩的だが暖かく、余白のある言葉遣い。しかしユーザーに提案する「セルフケア」は具体的で今日実践できる簡単な行動にしてください。
\${modeSpecificInstruction}

【安全設計（重要）】
自傷、希死念慮、他害、強い恐怖、パニック、現実との混同、暴力的なトラウマ想起が検知された場合、必ず 'risk_level' を 'high' に設定してください。
risk_levelが'high'の場合：
- 'dream_title' は「嵐のあとの静けさ」や「心が休まる場所」など、安全を感じられる穏やかなタイトルにしてください。
- 'summary' は簡潔にし、恐怖を煽らない言葉にしてください。
- 'metaphor' には「今は心が少し荒波の中にいる状態です」と書き、
- 'reflection_question' には「いま、深く息を吐いてみませんか？」など、グラウンディングを促す問いにしてください。
- 'inner_weather' の 'type' を「嵐」や「濃霧」にし、'description' で「今は夢の意味を深く考えるよりも、体を温めて、ゆっくり深呼吸して現実の安心感に触れることを最優先にしましょう。」と伝えてください。
- 'self_care' の 'action' を「深呼吸をして、あたたかい飲み物を飲みましょう」とし、'reason' を「心が休息と安全を必要としているサインだからです」としてください。
- 'garden_update' の 'new_place' は「静かな避難所」とし、'description' は「嵐をしのぎ、穏やかに安らぐためのあたたかいシェルターです」としてください。
```

## 5. UI/UX & デザインガイドライン (Visual & Styling)
* **テーマ**: **Deep Indigo & Slate Dark（深淵と星空のトーン）**。
  * ベースは深い夜空（`bg-slate-950`）、星や蛍の光を象徴するインディゴ、バイオレット、エメラルド、ゴールドのアクセントカラー。
* **タイポグラフィ**: 
  * メインフォント: **Inter** / システムサンセリフ（親しみやすさと洗練の同居）。
  * 装飾・ステータス: **JetBrains Mono** または **Space Grotesk** を極小サイズで使用し、洗練された「現像所（Dreamprint Lab）」の雰囲気を構築。
* **モーション**: `motion/react` を利用し、カードの展開や庭園のフェード、波打つパーティクルなど、優しく吸い込まれるようなイージング（`easeOut` / `easeInOut`）を設定。

## 6. 技術スタック (Technical Stack)
* **フロントエンド**: React 18 / Vite / TypeScript
* **スタイリング**: Tailwind CSS
* **アニメーション**: motion (from `motion/react`)
* **アイコン**: Lucide-react
* **バックエンド**: Express (Full-stack architecture running on port 3000)
* **データベース**: `data/db.json`（ファイルベースの軽量永続化ストレージ）
* **AI API**: Google GenAI SDK (`@google/genai` with `gemini-3.5-flash`)
